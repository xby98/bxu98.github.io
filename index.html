highlight:
    enable: false
prismjs:
    enable: false

如果使用 Hexo 7.0.0 之后的版本只需要修改为：

syntax_highlighter:

如果使用 Pandoc 还需要设置一下：

pandoc:
    extra:
        - no-highlight:

Markdown 简介
2024/10/8

Markdown 是一种轻量级的标记语言，设计初衷是以简单、易读易写的方式编写结构化的文档内容。它的语法规则非常直观，甚至在不转换的情况下也可以清晰地阅读，适合写作博客、文档、README 文件等。Markdown 编写的文件通常使用 .md 或 .markdown 后缀。

Markdown 的应用场景
技术文档：Markdown 被广泛应用于编写技术文档、项目说明等。
博客：如 Hexo、Jekyll 等静态网站生成器都支持 Markdown 语法。
协作平台：GitHub、GitLab 等平台的 README 文件通常使用 Markdown 编写，便于开发者之间的协作和交流。
Markdown 简单直观，非常适合需要快速编写和组织内容的场景。

常用语法示例

  1. 标题：使用 # 表示标题，# 的数量表示标题的级别。
# 一级标题
## 二级标题
### 三级标题
  1. 列表：
    无序列表：使用 -、* 或 +。
- 项目1
- 项目2
- 项目3

有序列表：使用数字加 .。

1. 第一项
2. 第二项
3. 第三项

  1. 粗体和斜体：
    粗体：**粗体** 或 __粗体__
    斜体：*斜体* 或 _斜体_
    粗斜体：***粗斜体***

  2. 链接：

[链接文本](https://example.com)
  1. 图片：
![图片描述](图片链接)
  1. 引用：使用 > 表示引用。
> 这是一个引用

7.
行内代码：使用反引号 ` 包裹。

这是 `inline code` 示例。

多行代码块：使用三个反引号 或缩进4个空格。

```inline code```

    inline code

| 头部1 | 头部2 |
|-------|-------|
| 内容1 | 内容2 |
| 内容3 | 内容4 |

我的第一篇博客-博客创建之路
2024/10/7

我搭建这个博客搜了好多教程，以下是我搭建的过程。

第一章 - 选择博客平台

可以选择使用现有的博客平台，或者自己搭建一个独立的博客。这里有几种常见的选择：

使用博客平台（简单、快速）
WordPress.com：不需要太多技术知识，可以快速创建和发布博客。平台提供托管服务，有免费和付费套餐。
Medium：适合想专注于写作而不关心技术搭建的人，提供简洁的界面，容易上手。
Blogger：Google 提供的免费博客平台，操作简单，适合初学者。
自托管博客（更多自定义、专业）
如果你想完全掌控自己的博客网站并能够进行更多的自定义，可以选择自托管方式。

WordPress.org：你需要购买域名和主机，并安装 WordPress。WordPress 是一个强大的内容管理系统（CMS），支持自定义主题和插件，灵活性很高。
Jekyll + GitHub Pages：Jekyll 是一个静态网站生成器，适合有一定技术背景的开发者。可以通过 GitHub Pages 免费托管，非常适合开发者展示技术博客。
Hexo：Hexo 是一个基于 Node.js 的静态博客生成器，速度快、简单易用，适合前端开发者使用。

第二章 - 选择域名和主机

如果你选择自托管博客，以下步骤至关重要：

购买域名：选择一个代表你品牌或个人风格的域名。可以使用域名注册商如 GoDaddy、Namecheap 等购买域名。
选择主机服务：购买适合你博客流量和内容需求的主机服务，如：
Bluehost：适合新手用户，简单易用。
SiteGround：WordPress 推荐的主机之一，速度和安全性较好。
GitHub Pages：适合使用 Jekyll 或 Hexo 搭建静态博客，提供免费的托管服务。
